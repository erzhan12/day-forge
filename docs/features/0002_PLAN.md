# Phase 2: Frontend Foundation (Inertia + Vue 3) — Technical Plan

Vue 3 + TypeScript frontend served via Vite, connected to Django 5.x using Inertia.js. The result: a working daily schedule page at `/schedule/{date}/` with full CRUD for time blocks, gap rendering, and a "now line" — no API layer, all data flows through Inertia page props.

---

## Phase 2A: Backend Wiring (Inertia Django Adapter)

### Dependencies

Add to `pyproject.toml`:
- `inertia-django` — Inertia.js server-side adapter for Django
- `whitenoise` — serves Vite-built static files in dev and prod

### `backend/day_forge/settings.py`

- Add `inertia` and `whitenoise.middleware.WhiteNoiseMiddleware` to `INSTALLED_APPS` and `MIDDLEWARE` respectively (WhiteNoise goes after `SecurityMiddleware`)
- Add `INERTIA_LAYOUT = "base.html"` — points to the single Django HTML shell
- Add `STATICFILES_DIRS = [BASE_DIR.parent / "frontend" / "dist"]` so Django can serve Vite output
- Add `STATIC_ROOT = BASE_DIR.parent / "staticfiles"` for `collectstatic`
- Add `INERTIA_SSR_ENABLED = False`

### `backend/day_forge/urls.py`

- Add `path("accounts/login/", schedules_views.login_view, name="login")`
- Add `path("accounts/logout/", schedules_views.logout_view, name="logout")`
- Add `path("schedule/<str:date>/", schedules_views.schedule_view, name="schedule")`
- Add `path("", schedules_views.root_redirect, name="root")` — redirects to today's schedule
- Add REST endpoints for TimeBlock CRUD (handled by a new `backend/schedules/api.py`):
  - `POST /api/schedules/<date>/blocks/` — create block
  - `PATCH /api/blocks/<pk>/` — update block (title, is_completed, category, times)
  - `DELETE /api/blocks/<pk>/` — delete block

### `backend/templates/base.html`

New file. The single HTML shell that Inertia requires. Loads Vite-built JS/CSS via `{% load static %}`. Must include:
- `{% block inertia %}{% endblock %}` (rendered by `inertia-django`)
- `<script type="module" src="{% static 'assets/app.js' %}"></script>`
- The Vite manifest is read by Django's static file loader; in dev, Vite runs separately on port 5173 with HMR and the `base.html` points to `http://localhost:5173/src/app.ts` instead

**Dev vs prod toggle**: use `DEBUG` setting — when `DEBUG=True`, `base.html` loads from Vite dev server; when `False`, loads from `dist/`.

### `backend/schedules/views.py`

All views use `inertia.render()` (from `inertia` package) instead of Django's `render()`.

**`root_redirect(request)`** — redirects to `/schedule/<today>/`

**`login_view(request)`**
- GET: `inertia.render(request, "Login", {})`
- POST: authenticate via `django.contrib.auth.authenticate` + `login`; on success redirect to `/schedule/<today>/`; on failure re-render with `{"errors": {"non_field": "Invalid credentials"}}`
- Wrap with `@require_http_methods(["GET", "POST"])`

**`logout_view(request)`** — POST only, calls `auth.logout(request)`, redirects to `/accounts/login/`

**`schedule_view(request, date)`**
- Parse `date` string as `datetime.date`; return 404 on bad format
- `get_or_create` a `Schedule` for that date
- Query `TimeBlock.objects.filter(schedule=schedule).order_by("start_time", "sort_order")`
- Serialize blocks to list of dicts (id, title, start_time as "HH:MM", end_time as "HH:MM", category, is_completed, sort_order)
- Return `inertia.render(request, "Schedule", {"schedule": {...}, "blocks": [...], "date": str(date)})`
- Wrap with `@login_required`

### `backend/schedules/api.py`

Django views that handle JSON mutations. All require `@login_required` and `@csrf_exempt` is NOT used — CSRF token is sent by Inertia automatically via `X-XSRF-TOKEN` header; configure `CSRF_COOKIE_NAME = "XSRF-TOKEN"` and `CSRF_HEADER_NAME = "HTTP_X_XSRF_TOKEN"` in settings.

**`create_block(request, date)`** — POST, creates `TimeBlock` linked to the schedule for `date`, returns JSON of created block

**`update_block(request, pk)`** — PATCH, partial update of `TimeBlock` fields (title, is_completed, start_time, end_time, category), returns updated block JSON

**`delete_block(request, pk)`** — DELETE, removes `TimeBlock`, returns `{"ok": true}`

All three validate input and return `400` with `{"errors": {...}}` on failure.

### `backend/tests/test_views.py` (new file)

- Test `schedule_view` returns 302 for unauthenticated requests
- Test `schedule_view` returns Inertia response with `blocks` prop for authenticated user
- Test `create_block` creates a `TimeBlock` and returns its JSON
- Test `update_block` with `is_completed=true` updates the record
- Test `delete_block` removes the record

## Phase 2C: Vue Pages & Components

### `src/pages/Login.vue`

- Receives Inertia prop: `errors: { non_field?: string }`
- Form with username + password fields
- On submit: `router.post("/accounts/login/", { username, password })` using `@inertiajs/vue3`'s `useForm`
- Shows `errors.non_field` if present

### `src/pages/Schedule.vue`

Receives Inertia props:
- `schedule: Schedule`
- `blocks: TimeBlock[]`
- `date: string`

Renders in order:
1. `<DateNavigator :date="date" />`
2. `<NowLine>` (only when `date === today`)
3. Interleaved list of `<TimeBlock>` and `<GapSlot>` components built by the gap-rendering algorithm (see below)
4. `<AddBlockForm>` always visible at bottom (or revealed via button)

Uses `useSchedule` composable for all data mutations.

### Gap-rendering algorithm (in `Schedule.vue` or `useSchedule.ts`)

Given `blocks: TimeBlock[]` sorted by `start_time`:

1. Define `DAY_START = "00:00"` and `DAY_END = "23:59"`
2. Build a `renderList: Array<{type: "block"|"gap", ...}>`:
   - If `blocks[0].start_time > DAY_START`, prepend a gap from `DAY_START` to `blocks[0].start_time`
   - For each consecutive pair `(blocks[i], blocks[i+1])`: if `blocks[i].end_time < blocks[i+1].start_time`, insert a gap between them
   - After last block: if `blocks[last].end_time < DAY_END`, append trailing gap
3. Each gap entry carries: `{ start_time, end_time, duration_minutes }`

`duration_minutes` computed as: `(end_time_mins - start_time_mins)` where each time is converted to total minutes.

### `src/components/DateNavigator.vue`

- Props: `date: string` (YYYY-MM-DD)
- Displays formatted date (e.g., "Sunday, April 6")
- "‹" button: `router.visit("/schedule/" + prevDate)`
- "›" button: `router.visit("/schedule/" + nextDate)`
- "Today" button: `router.visit("/schedule/" + today)`
- Prev/next date computed by parsing `date` prop and offsetting by ±1 day

### `src/components/TimeBlock.vue`

Props: `block: TimeBlock`

Emits: `updated`, `deleted`

Visual structure:
- Left color bar: 4px wide, colored by `category` (work=#3B82F6, personal=#8B5CF6, health=#10B981, other=#6B7280)
- Time badge: `"HH:MM – HH:MM"` formatted string
- Duration indicator: computed as `(end_time - start_time)` in minutes, displayed as `"Xm"` or `"Xh Ym"`
- Checkbox: clicking sends `PATCH /api/blocks/:id/` with `{is_completed: !block.is_completed}` via `useSchedule.updateBlock`
- Title: display mode shows `<span>` with title text; clicking switches to `<input>` (inline edit)

**Inline title editing**:
1. On click of title span, set `editing = true`, render `<input :value="block.title">`
2. On `blur` or `keydown.enter`: call `useSchedule.updateBlock(block.id, { title: inputValue })`; set `editing = false`
3. On `keydown.escape`: revert to original title, set `editing = false`

**Delete button**: small "×" button, on click shows `window.confirm("Delete this block?")`, on confirm calls `useSchedule.deleteBlock(block.id)`

### `src/components/GapSlot.vue`

Props: `startTime: string`, `endTime: string`, `durationMinutes: number`

Displays: `"Free — Xh Ym"` (or just `"Xm"` if under 60 min)

Click handler: emits `add-here` event with `{ start_time, end_time }` — `Schedule.vue` listens and pre-fills `AddBlockForm` with those times

### `src/components/AddBlockForm.vue`

Props: `initialStartTime?: string`, `initialEndTime?: string`

Fields: title (text), start_time (time input, step=300 for 5-min increments), end_time (time input), category (select)

On submit: calls `useSchedule.createBlock({ title, start_time, end_time, category })`; clears form on success

### `src/components/NowLine.vue`

- Computes current time as minutes since midnight: `now.getHours() * 60 + now.getMinutes()`
- Positions a horizontal red line within a scroll container using `top: (nowMinutes / 1440 * 100)%` or via a pixel mapping consistent with block heights
- Sets up `setInterval` every 60 seconds to recompute
- On mount: calls `element.scrollIntoView({ behavior: "smooth", block: "center" })`
- Cleans up interval on `onUnmounted`

### `src/composables/useSchedule.ts`

Wraps all mutations using `@inertiajs/vue3`'s `router` (Inertia visits reload page props automatically after each mutation):

**`createBlock(data)`**: `router.post("/api/schedules/${date}/blocks/", data)`

**`updateBlock(id, data)`**: `router.patch("/api/blocks/${id}/", data)`

**`deleteBlock(id)`**: `router.delete("/api/blocks/${id}/")`

After each mutation Inertia re-fetches the schedule view and merges updated `blocks` prop into the page — no local state management needed for the base phase.
