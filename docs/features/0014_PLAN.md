# Feature 0014 â€” Bug Fix: `ai_command` crashes on non-dict JSON body

## Context

`POST /api/ai/schedules/<date>/command/` returns HTTP 500 when the request body is valid JSON whose root is not an object (e.g. `[]`, `"x"`, `123`, `null`, `true`). The endpoint parses the body with `json.loads` but immediately calls `data.get("command")` without first verifying that `data` is a `dict`, so a non-dict root raises `AttributeError` and falls through to an unhandled 500.

The sibling endpoint `ai_chat` (`backend/ai/views.py`, `async def ai_chat`, ~line 947) already has the exact same guard â€” `if not isinstance(data, dict)` â€” immediately after its `json.loads` call, making `ai_command` the inconsistent outlier. `ai_generate_draft` does not parse a JSON body and is unaffected.

This plan documents the fix shipped in PR #37 (branch `cursor/critical-correctness-bugs-5519`) so it can be tracked as a numbered feature record and implemented independently (the cursor-bot PR is not being merged).

---

## Root Cause

`backend/ai/views.py` â€” `async def ai_command` (~line 534):

```
data = json.loads(request.body)
# missing isinstance guard here
command = data.get("command")   # AttributeError if data is not a dict
```

The `json.loads` try/except only catches `json.JSONDecodeError` (returning a 400 "Invalid JSON."). Any other JSON root type bypasses that error path and raises `AttributeError` on the next line, resulting in a 500.

---

## Fix

### `backend/ai/views.py`

In `async def ai_command`, immediately after the `json.loads` try/except block (currently ~line 534), add a dict-type guard matching the pattern already used in `ai_chat`:

- If `data` is not a `dict`, return a `JsonResponse` with status **400** and the body:
  `{"errors": {"body": "Request body must be a JSON object."}}`

The error envelope shape `{"errors": {"body": "..."}}` is consistent with:
- The existing `{"errors": {"body": "Invalid JSON."}}` response immediately above it in the same function.
- The identical guard in `ai_chat` (same envelope, same status).

No other functions in this file require changes.

### `backend/tests/test_ai_views.py`

Add a parametrized regression test inserted at ~line 65. The test should be parametrized over the five non-dict JSON literals:

- `"[]"`
- `'"x"'`
- `"123"`
- `"null"`
- `"true"`

Each parametrized case must assert:
1. HTTP status code **400**.
2. The exact error string `"Request body must be a JSON object."` in the response JSON.

The test slots into the existing test class that covers `ai_command` input-validation paths.
