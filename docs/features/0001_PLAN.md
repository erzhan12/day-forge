# Phase 1: Project Scaffold & Data Layer — Technical Plan

Django 5.x project with SQLite data layer, Docker dev environment, and admin access for all models. No frontend. The result: `docker compose up` serves Django on :8000 with full admin CRUD for all 6 models.

---

## Sub-phase 1A: pyproject.toml & Project Bootstrap

### pyproject.toml updates
- Add dependencies: `django>=5.0,<6.0`
- Add dev dependencies group: `pytest`, `pytest-django`, `ruff`
- Configure `[tool.ruff]` section (line-length, target Python 3.14)
- Configure `[tool.pytest.ini_options]` with `DJANGO_SETTINGS_MODULE = "day_forge.settings"`
- Package manager: `uv` (all commands via `uv run`, `uv add`, `uv sync`)

### Django project init
- Run `django-admin startproject day_forge backend/` to create the project root inside `backend/`
- This creates `backend/day_forge/` (settings, urls, wsgi, asgi) and `backend/manage.py`

### Create 4 Django apps inside `backend/`
- `uv run python backend/manage.py startapp schedules` → move to `backend/schedules/`
- `uv run python backend/manage.py startapp templates_mgr` → move to `backend/templates_mgr/`
- `uv run python backend/manage.py startapp analytics` → move to `backend/analytics/`
- `uv run python backend/manage.py startapp ai` → move to `backend/ai/`

## Sub-phase 1B: Django Settings & Database Configuration

### `backend/day_forge/settings.py`
- Register apps in `INSTALLED_APPS`: `schedules`, `templates_mgr`, `analytics`, `ai`, plus `django.contrib.admin`, `django.contrib.auth`, etc.
- `DATABASES`: SQLite pointing to `db/day_forge.db` (relative to project root, outside `backend/` so Docker can volume-mount it)
- SQLite WAL mode: add `"OPTIONS": {"init_command": "PRAGMA journal_mode=WAL;"}` — **note**: Django SQLite backend doesn't support `init_command` natively. Instead, use a `connection_created` signal handler in `day_forge/__init__.py` or `settings.py` that executes `PRAGMA journal_mode=WAL; PRAGMA foreign_keys=ON;`
- `AUTH_USER_MODEL`: use Django's default `auth.User` (no custom user model needed for single-user)
- Set `LANGUAGE_CODE`, `TIME_ZONE` as appropriate
- `ALLOWED_HOSTS = ["*"]` for local dev (tighten in prod)

### WAL mode setup — `backend/day_forge/__init__.py`
- Connect to `django.db.backends.signals.connection_created`
- In the handler, execute `PRAGMA journal_mode=WAL;` and `PRAGMA foreign_keys=ON;` on the connection cursor

## Sub-phase 1C: Models

All models use Django's auto-incrementing `id` as primary key. Timestamps (`created_at`, `updated_at`) added via `auto_now_add` / `auto_now` where useful.

### `schedules` app — `backend/schedules/models.py`

**Schedule**
- `date` — `DateField(unique=True)` — one schedule per day
- `status` — `CharField(max_length=10, choices=[("draft","Draft"), ("active","Active"), ("reviewed","Reviewed")], default="draft")`
- `created_at` — `DateTimeField(auto_now_add=True)`
- `updated_at` — `DateTimeField(auto_now=True)`
- `__str__` returns `f"{self.date} ({self.status})"`
- `Meta.ordering = ["-date"]`

**TimeBlock**
- `schedule` — `ForeignKey(Schedule, related_name="time_blocks", on_delete=CASCADE)`
- `title` — `CharField(max_length=255)`
- `start_time` — `TimeField()` — 5-minute granularity enforced at validation level (see below)
- `end_time` — `TimeField()` — same constraint
- `category` — `CharField(max_length=10, choices=[("work","Work"), ("personal","Personal"), ("health","Health"), ("other","Other")], default="other")`
- `is_completed` — `BooleanField(default=False)`
- `sort_order` — `IntegerField(default=0)` — ordering tiebreaker within same time
- `created_at` — `DateTimeField(auto_now_add=True)`
- `Meta.ordering = ["start_time", "sort_order"]`

**5-minute granularity validation**: override `clean()` on `TimeBlock` to validate that both `start_time.minute % 5 == 0` and `end_time.minute % 5 == 0`, and that `start_time < end_time`. Register this in admin via `clean()` call path. Also add a `validators` list on the fields pointing to a shared `validate_five_minute_granularity(value)` function in `backend/schedules/validators.py`.

### `templates_mgr` app — `backend/templates_mgr/models.py`

**Template**
- `name` — `CharField(max_length=100)`
- `type` — `CharField(max_length=10, choices=[("weekday","Weekday"), ("weekend","Weekend")])`
- `blocks` — `JSONField(default=list)` — array of dicts: `{"title": str, "start_time": "HH:MM", "end_time": "HH:MM", "category": str}`
- `created_at` — `DateTimeField(auto_now_add=True)`
- `updated_at` — `DateTimeField(auto_now=True)`
- `__str__` returns `f"{self.name} ({self.type})"`

**Rule**
- `text` — `TextField()` — natural language rule
- `is_active` — `BooleanField(default=True)`
- `priority` — `IntegerField(default=0)` — higher = more important
- `created_at` — `DateTimeField(auto_now_add=True)`
- `Meta.ordering = ["-priority"]`

### `ai` app — `backend/ai/models.py`

**AIInteraction**
- `schedule` — `ForeignKey("schedules.Schedule", related_name="ai_interactions", on_delete=CASCADE)`
- `user_command` — `TextField()`
- `ai_response` — `TextField()`
- `actions_json` — `JSONField(default=list)` — parsed actions that were applied
- `created_at` — `DateTimeField(auto_now_add=True)`

### `analytics` app — `backend/analytics/models.py`

**DailyReview**
- `schedule` — `OneToOneField("schedules.Schedule", related_name="daily_review", on_delete=CASCADE)`
- `planned_count` — `IntegerField(default=0)`
- `completed_count` — `IntegerField(default=0)`
- `skipped_count` — `IntegerField(default=0)`
- `notes` — `TextField(blank=True, default="")`
- `created_at` — `DateTimeField(auto_now_add=True)`

## Sub-phase 1G: Docker Setup

### `Dockerfile`
- Base image: `python:3.14-slim`
- Install `uv` via pip or official install script
- Copy `pyproject.toml` and `uv.lock` first (layer caching)
- `uv sync` to install deps
- Copy `backend/` source
- `WORKDIR /app/backend`
- `CMD ["uv", "run", "python", "manage.py", "runserver", "0.0.0.0:8000"]`

### `docker-compose.yml`
- Single service: `web`
  - Build from `.` (project root Dockerfile)
  - Ports: `8000:8000`
  - Volumes:
    - `./backend:/app/backend` — hot-reload (Django dev server auto-reloads on file changes)
    - `./db:/app/db` — SQLite database persisted on host
  - Environment: `DJANGO_SETTINGS_MODULE=day_forge.settings`, `DEBUG=1`
  - Command override: `uv run python manage.py runserver 0.0.0.0:8000`

### Startup sequence (documented in README or compose comments)
1. `docker compose build`
2. `docker compose run web uv run python manage.py migrate`
3. `docker compose run web uv run python manage.py createsuperuser`
4. `docker compose run web uv run python manage.py seed_templates`
5. `docker compose up`
