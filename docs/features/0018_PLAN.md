# 0018 — Suppress undo toast for obvious in-UI schedule edits

## Context

The bottom-right undo toast (`UndoToast.vue`) currently fires after **every**
successful schedule mutation via `pushUndo` in
`frontend/src/composables/useUndo.ts`. For manual in-UI edits — adding a block,
deleting via the X button, inline rename, check/uncheck, drag reorder — the
result is already visible on the timeline and trivially reversible by hand, so
the toast is redundant noise.

This feature adds an opt-in `silent` flag to `UndoAction`. Silent actions are
still pushed onto the undo stack (Cmd+Z / Ctrl+Z keeps working) but skip the
toast popup. The two non-obvious mutations (AI chat apply, generate draft) and
all of `performUndo`'s own toasts (undo result, errors, "Nothing to undo.")
stay unchanged.

GitHub Issue #54.

## Scope (verbatim)

| Action | Toast today | Desired | Call site |
|--------|-------------|---------|-----------|
| Add block | Yes | Silent | `AddBlockForm.vue` (`type: "add"`) |
| Delete via X | Yes | Silent | `TimeBlock.vue` (`type: "delete"`) |
| Rename | Yes | Silent | `TimeBlock.vue` (`type: "edit"`) |
| Check / uncheck | Yes | Silent | `TimeBlock.vue` (`type: "toggle"`) |
| Drag reorder | Yes | Silent | `useDrag.ts` (`type: "drag"`) |
| AI chat apply | Yes | **Keep** | `useChat.ts` (`type: "ai"`) |
| Generate draft | Yes | **Keep** | `Schedule.vue` (`type: "draft"`) |
| Cmd+Z / undo result / errors | Yes | **Keep** | `performUndo` (no `pushUndo`) |

## Algorithm

`pushUndo(action)`:
1. `undoStack.value.push(action)` (unchanged).
2. If `undoStack.value.length > MAX_UNDO_STACK`, `shift()` (unchanged).
3. **New:** if `!action.silent`, call `showToast(action.description, true)`.
   Otherwise skip — stack updated silently.

Silent actions therefore remain at the top of the stack and are restored
normally by `performUndo`, which still surfaces its own `Undone: …` /
"Undo failed. …" toast regardless of the originating action's `silent` flag.

## Acceptance criteria (verbatim)

- Adding a block shows no toast; Cmd+Z still restores the pre-add state.
- Deleting, renaming, toggling, dragging a block: same — no toast, undo still works.
- AI chat apply and generate-draft still show the toast with an Undo link.
- Cmd+Z on an empty stack still shows "Nothing to undo."
- An undo network failure still shows the error toast.
